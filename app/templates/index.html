Security Twitter Lab

Demonstration of RBAC + DAC security model implementation

This application implements 4 roles with different permissions:

  • Main Admin - Can do anything
  • Group Admin - Full control within assigned groups
  • User - Can create, edit, delete own posts, transfer ownership
  • Guest - Can only view posts

RBAC

Role-Based Access Control. Users are assigned roles (Main Admin, Group Admin, User, Guest) that determine their base permissions.

DAC

Discretionary Access Control. Post owners can transfer ownership to other users. Group admins own specific groups.

Policy Pattern

All permission checks are centralized in the Policy class, making security rules easy to understand and maintain.
